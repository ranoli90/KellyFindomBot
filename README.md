# KellyFindomBot

**A Telegram userbot that runs the Kelly Financial Dominatrix persona — adaptive per-user, tribute-gated, and fully self-hosted.**

KellyFindomBot is a Telegram userbot (MTProto via Telethon) that operates as a real Telegram user account, not a bot. It runs a psychological findom dynamic with per-user persona adaptation, a tribute gate enforced via Telegram Stars, and a local LLM backend. No cloud AI APIs, no subscriptions.

---

## What It Does

Kelly is a Financial Dominatrix persona that:

- **Responds to every user differently** — the first message is always adapted to what they actually said, never a canned intro. As the conversation grows, Kelly's tone, references, and engagement style adapt to what keeps that specific sub invested.
- **Enforces tribute before real engagement** — FREE-tier users get up to 3 dominant responses directing them to pay $50 via Telegram Stars, then silence. Payment is handled by a companion Bot API bot (`PAYMENT_BOT_TOKEN`), so the main userbot never touches money.
- **Classifies user intent in real-time** — each message is classified (READY / HIGH_VALUE / WINDOW_SHOPPER / TIME_WASTER / TESTER / PROMISE_TO_PAY / POSITIVE_CONFIRM) and Kelly's response adapts accordingly.
- **Re-engages dormant conversations** — users who haven't replied in 2–21 days get a personalized callback message generated by the LLM. Max 2 per day, with FloodWait safety built in.
- **Never introduces herself as Kelly** — no name/bio dump in message one. The persona surfaces naturally through conversation.
- **Keeps payments anonymous** — only Stars and crypto are accepted. Both are anonymous by design. Stars explanation: "you buy them directly in the Telegram app — same as any in-app purchase."

## Quick Start (Local)

### Prerequisites

- Python 3.11+
- A Telegram account with API credentials from [my.telegram.org/apps](https://my.telegram.org/apps)
- A local LLM server (llama-server recommended — see below)
- `PAYMENT_BOT_TOKEN` from BotFather (required for Stars tribute)

### 1. Install

```bash
git clone https://github.com/ranoli90/KellyFindomBot
cd KellyFindomBot
pip install -r requirements.txt
cp .env.example .env
# Edit .env with your credentials
```

### 2. Configure `.env`

```env
TELEGRAM_API_ID=your_api_id
TELEGRAM_API_HASH=your_api_hash
ADMIN_USER_ID=your_numeric_telegram_user_id

BOT_PERSONA=kelly
ENABLE_MONETIZATION=true

PAYMENT_BOT_TOKEN=123456:ABC-your-payment-bot-token
PAYMENT_BOT_USERNAME=YourTributeBot

ELEVENLABS_API_KEY=        # optional — for voice notes
ELEVENLABS_VOICE_ID=       # optional
MONITOR_AUTH_TOKEN=change_me_now
```

### 3. Start the LLM backend

```bash
# Recommended: Dolphin 2.9.3 Mistral-Nemo-12B (12B, uncensored)
llama-server \
  -m /path/to/Dolphin-2.9.3-Mistral-Nemo-12B-Q6_K.gguf \
  --host 0.0.0.0 --port 1234 \
  -ngl 99 -c 32768

# Verify it's up
curl http://localhost:1234/v1/models
```

Any OpenAI-compatible endpoint works — LM Studio, Ollama (with OpenAI compat), or a cloud API.

### 4. Run the bot

```bash
# First run — prompts for phone number and verification code
python kelly_telegram_bot.py --personality kelly_persona.yaml --small-model --monitoring

# Subsequent runs — uses saved session
python kelly_telegram_bot.py --personality kelly_persona.yaml --small-model
```

Session is saved as `kelly_session.session`. Protect this file — it is your Telegram account credential.

## Tribute / Payment Flow

1. User sends `/start` or first message → Kelly responds with a dominant frame-setting opener + Stars invoice
2. User pays via the Telegram Stars invoice (handled by `PAYMENT_BOT_TOKEN` bot)
3. Payment event updates the user's tier from FREE → PAID in `user_profiles/`
4. On next message, `get_access_tier()` returns PAID → gate never fires → LLM responds with full Kelly persona

**Stars:** Built into Telegram. Users buy Stars in the Telegram app or iOS App Store, then tap the invoice link. No third-party accounts needed.

**Crypto:** Supported conversationally — Kelly asks what they have and arranges manually. No automated crypto infrastructure in this codebase.

**Tribute amounts:**
- `ACCESS_TIER_FAN_THRESHOLD = 2500 Stars` ≈ $50 → unlocks full access
- `KELLY_MIN_TRIBUTE_STARS = 500 Stars` ≈ $10 → also unlocks (flexible minimum)
- `VERIFY_STARS = 125` ≈ $2.50 → verification photo

## Telegram Account Safety (2026)

KellyFindomBot takes the following steps to protect the account from bans and restrictions:

- **Device fingerprint**: `TelegramClient` is initialized with `device_model="iPhone 15 Pro"`, matching a real mobile client.
- **Flood sleep threshold**: Auto-sleeps up to 60s for FloodWait errors instead of crashing.
- **Re-engagement rate limit**: Max 2 proactive messages per day, 2–5 minutes between sends.
- **FloodWaitError handling**: Re-engagement loop catches FloodWaitError, sleeps the required time + buffer.
- **PeerFloodError handling**: Pauses all re-engagement for 24 hours. This error means Telegram is rate-limiting the account for sending to new users.
- **Account ban detection**: `UserDeactivatedBanError` / `PhoneNumberBannedError` → re-engagement stops entirely + critical log alert.
- **Session backup**: `AuthKeyUnregisteredError` triggers session restore from backup before requiring manual re-auth.
- **Single session**: Never run two Telethon processes with the same session file simultaneously.

**Additional operator precautions** (not automated — your responsibility):
- Use a phone number that has 2FA enabled
- Keep the account warm: log in from the actual phone occasionally
- Don't send the same message text to many users — vary all outgoing messages (the LLM ensures this)
- Don't run other bots or scrapers from the same account

## Known Limitations

- **Single session**: Only one Telethon process can use the session file at a time.
- **LLM hallucinations**: Small models (12B) occasionally invent backstory. Post-processing catches most but not all.
- **No automated crypto**: Crypto payments are handled manually by operator — no wallet addresses or confirmation flow.
- **Stars only (automated)**: Full tribute automation only works with Telegram Stars + `PAYMENT_BOT_TOKEN`.
- **Re-engagement for free-tier users**: The re-engagement system targets users who had conversations. FREE-tier users who bounced without ever paying are not automatically re-targeted (by design — would increase spam risk).
